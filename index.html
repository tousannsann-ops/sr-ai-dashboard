03流量分布问题
Google network/search/youtube · Meta 待补充
先按 Google / Meta 建立版位诊断框架：Google 重点看 network / search / youtube 的流量结构与 CTR / CVR 断点；Meta 暂无 AI 对比数据，先保留 IG / reels / feed 的占比假设与异常排查方向。
Google 版位分析框架 network 50-60% · search 10% · youtube 20-30%
先看版位结构是否接近 network 50-60%、search 10%、youtube 20-30%；再拆 CTR / CVR 异常究竟来自素材适配、关键词意图、版位承接还是算法扩量。
CTR 异常诊断
network 看素材是否过泛、低质 app 面是否拉低点击；search 看关键词意图与文案是否错配；youtube 看首屏 3 秒钩子、封面和受众包是否失效。
CVR 异常诊断
network 低 CVR 先排查误点流量和商店页承接；search 低 CVR 看高点击关键词是否过宽；youtube 低 CVR 看视频承诺与下载页体验是否不一致。
Meta 版位分析框架 暂无 AI 对比数据，待补充
先保留版位框架：IG 占比 >90%，reels + 激励视频 50-60%，feed 30%。待补齐 AI / 非AI 分版位消耗、CTR、CVR 后，再验证异常到底出在库存结构还是创意匹配。
CTR 异常诊断
IG 整体低 CTR 看封面、首帧和文案钩子；reels / 激励视频低 CTR 看短视频节奏与互动设计；feed 低 CTR 看主视觉、标题和评论区社证是否不足。
CVR 异常诊断
reels / 激励视频高点击低转化时先排查误触和低意图流量；feed 低 CVR 看受众过宽、落地页承接和转化事件回传是否失真。
